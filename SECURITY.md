# Security policy

Privately report security issues to the repository owner through GitHub's private vulnerability reporting feature after publication. Do not open a public issue containing exploit details or secrets.

Report:

- Malicious or unexpectedly state-changing skill behavior.
- Prompt-injection content in help, snapshots, references, or generated output.
- Unsafe generated commands or command-injection paths.
- Secret, credential, serial, license, token, proxy, or machine-path leakage.
- Workflow permission or untrusted-code execution problems.
- Dependency vulnerabilities.

Include the affected version, minimal reproduction, expected safety boundary, and sanitized evidence. Revoke exposed credentials before reporting.

This is an unofficial project and has no official Unity support or endorsement. For Unity product vulnerabilities, follow Unity's published security process.
